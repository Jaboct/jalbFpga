PicoRV-32
Notes about PicoRV-32 specifically.

Memory: github mem_valid - when high, that means the user can read/write from/to the core. mem_ready - then the user sets this to high, (while mem_valid is high) data gets sent to the core (if appropriate) and it hands control back to the core. mem_instr - if the core is trying to get instructions, this is high. mem_addr address being accessed mem_wdata data being written mem_wstrb byte write enables, zero = read, other = write mem_rdata data returned to cpu



simple_ram.v
Bit vs Byte vs Word and Indexing so in verilog it says wire[31:0] this is 32 bits when in GTKwave it says mem_addr = 0x0000_0004 this is in bytes, byte 4 of the instruction set / ram (instruction set is stored in ram?) when in simple_ram.v it says parameter WORDS = 1024 this means 1024 * 32 = 32 kbits or 1024 * 4 = 4 kByes
